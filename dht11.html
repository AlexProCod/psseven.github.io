Подключение датчика DHT11 / DHT22 к ESP8266

Датчики серии DHT (DHT11 и DHT22/AM2302) — это самые популярные цифровые сенсоры для измерения температуры и влажности в проектах на Arduino и ESP8266. Они дешевы, просты в подключении и имеют встроенный АЦП, что избавляет микроконтроллер от лишней нагрузки. В этой статье мы разберем отличия моделей, схему подключения и работу с библиотекой.

В чем разница: DHT11 vs DHT22?

Внешне модули выглядят похоже (синий корпус у DHT11, белый у DHT22), но их характеристики существенно различаются.

⚖️ Сравнение характеристик
Параметр DHT11 (Синий) DHT22 (Белый)
Температура 0...50°C (±2°C) -40...80°C (±0.5°C)
Влажность 20...90% (±5%) 0...100% (±2-5%)
Частота опроса 1 раз в секунду 1 раз в 2 секунды
Цена Дешевле Немного дороже

Рекомендация: Для уличных метеостанций и точных измерений однозначно выбирайте DHT22. DHT11 подойдет для простых учебных проектов или контроля климата в помещении, где не нужна высокая точность.

Что вам понадобится

  • Микроконтроллер ESP8266 (NodeMCU, Wemos D1 Mini).
  • Датчик DHT11 или DHT22 (в виде модуля с платой или отдельный компонент).
  • Резистор 10 кОм (нужен только для голого датчика без платы).
  • Соединительные провода.
  • Компьютер с Arduino IDE.

Схема подключения

Здесь важно различать два варианта исполнения датчика:

Вариант 1: Модуль с платой (3 контакта)

Если у вас датчик уже припаян к маленькой плате с надписью DHT11/DHT22 и тремя пинами, то подтягивающий резистор уже установлен на плате. Подключать просто:

  • VCC → 3V3 (или 5V)
  • GND → GND
  • DATA → Любой GPIO (например, D4)

Вариант 2: Голый датчик (4 контакта)

Если вы купили отдельный компонент в пластиковом корпусе с 4 ножками, вам обязательно нужен подтягивающий резистор.

⚡
❗ Важно: Подтягивающий резистор
Между пином VCC и пином DATA необходимо подключить резистор номиналом 10 кОм. Без него связь с датчиком будет нестабильной.
Пин датчика (слева направо) Назначение Подключение к ESP8266
1 VCC (Питание) 3V3 (или 5V)
2 DATA (Данные) GPIO (напр. D4) + Резистор 10к к VCC
3 NC (Не подключать) Оставить висеть в воздухе
4 GND (Земля) GND

Настройка Arduino IDE

Для работы с датчиками DHT необходима библиотека от Adafruit. Установите две библиотеки через Sketch > Include Library > Manage Libraries:

  1. DHT sensor library (автор: Adafruit).
  2. Adafruit Unified Sensor (зависимость, установится автоматически или найдите отдельно).

Пример кода

Универсальный скетч, который работает и с DHT11, и с DHT22. Нужно лишь раскомментировать нужную строку в начале кода.

#include 

// НАСТРОЙКИ
#define DHTPIN D4     // Пин, к которому подключен DATA
//#define DHTTYPE DHT11   // Раскомментируйте для DHT11
#define DHTTYPE DHT22   // Раскомментируйте для DHT22

DHT dht(DHTPIN, DHTTYPE);

void setup() {
  Serial.begin(115200);
  Serial.println("Запуск датчика DHT...");
  dht.begin();
}

void loop() {
  delay(2000); // Задержка между измерениями (минимум 2 сек для DHT22)

  // Считывание влажности
  float h = dht.readHumidity();
  // Считывание температуры в Цельсиях
  float t = dht.readTemperature();
  // Считывание температуры в Фаренгейтах
  float f = dht.readTemperature(true);

  // Проверка на ошибки чтения
  if (isnan(h) || isnan(t) || isnan(f)) {
    Serial.println("Ошибка чтения данных с датчика DHT!");
    return;
  }

  // Вычисление индекса жары (теплоощущение)
  float hif = dht.computeHeatIndex(f, h);
  // Или в Цельсиях
  float hic = dht.computeHeatIndex(t, h, false);

  Serial.print("Влажность: ");
  Serial.print(h);
  Serial.print("% | Температура: ");
  Serial.print(t);
  Serial.print(" °C | Теплоощущение: ");
  Serial.print(hic);
  Serial.println(" °C");
}
💡 Важные нюансы кода
  • Задержка: Обязательно делайте паузу delay(2000) или больше между запросами. Датчик медленный и не успеет обновить данные при частых опросах.
  • Проверкаisnan(): Функция read... может вернуть NaN (Not a Number), если чтение не удалось. Всегда проверяйте это перед использованием данных.
  • Питание: Если датчик работает нестабильно, попробуйте подать на VCC 5В вместо 3.3В (датчик допускает 3-6В), но сигнальный пин DATA оставьте на 3.3В логике ESP8266.

Возможные проблемы

  • Чтение возвращает NaN:
    • Проверьте наличие подтягивающего резистора 10 кОм (если используете голый датчик).
    • Убедитесь, что выбран правильный тип датчика в коде (DHT11 или DHT22).
    • Проверьте надежность контактов.
  • Данные не обновляются: Вы делаете опрос слишком часто. Увеличьте интервал в loop() до 2000 мс.
  • Неверные показания влажности: Держите датчик подальше от источников тепла (сам ESP8266 может нагреваться) и влаги. Обеспечьте циркуляцию воздуха вокруг сенсора.

Применение в умном доме

Датчики DHT идеально подходят для:

  • Мониторинга микроклимата в комнате с выводом данных на OLED-дисплей или в Telegram.
  • Управления увлажнителем воздуха или осушителем по пороговым значениям.
  • Контроля условий в теплице или инкубаторе.
  • Ведения логов погоды на домашнем сервере (Home Assistant, Domoticz).

Заключение

DHT11 и DHT22 — это «рабочие лошадки» мира любительской электроники. Несмотря на появление более точных сенсоров (типа BME280), они остаются популярными благодаря своей дешевизне и простоте. Правильное подключение подтягивающего резистора и соблюдение временных интервалов опроса гарантируют стабильную работу вашего проекта.
