Система умного дома на ESP8266 с Telegram

В этом проекте мы превратим локальную систему мониторинга в полноценную систему безопасности умного дома. ESP8266 будет отслеживать открытие двери, движение и температуру, а при возникновении тревожных событий мгновенно отправлять уведомления вам в Telegram. Это позволяет контролировать жилье из любой точки мира.

Архитектура системы

Система работает по следующему алгоритму:

  1. ESP8266 подключается к Wi-Fi.
  2. В цикле опрашиваются датчики (Геркон, PIR, DHT11/22).
  3. При изменении состояния (дверь открылась, обнаружено движение) формируется HTTP-запрос к API Telegram.
  4. Бот отправляет сообщение в ваш чат.
🔒 Безопасность Данные передаются по защищенному протоколу HTTPS. Ваш токен бота должен храниться в секрете и не выкладываться в открытый доступ (GitHub и т.д.).

Необходимые компоненты

  • ESP8266 (NodeMCU или Wemos D1 Mini).
  • Датчик открытия двери (Геркон).
  • Датчик движения (AM312 или HC-SR501).
  • Датчик температуры и влажности (DHT11 или DHT22).
  • Резисторы (4.7 кОм для DHT, 10 кОм для геркона - опционально).
  • Смартфон с установленным Telegram.

Шаг 1: Настройка Telegram бота

🤖 Создание бота и получение токена

  1. Откройте Telegram и найдите бота @BotFather.
  2. Отправьте команду /newbot.
  3. Придумайте имя боту (например, MyHomeSecurityBot) и юзернейм (должен заканчиваться на bot, например my_home_sec_bot).
  4. BotFather выдаст вам API Token (длинная строка символов). Сохраните его, он понадобится для кода.

🆔 Получение Chat ID

  1. Найдите своего нового бота в поиске и нажмите Start (или отправьте /start).
  2. Откройте бота @userinfobot (или любого аналога) и нажмите Start.
  3. Он пришлет вам ваш Id (набор цифр). Это ваш Chat ID.
  4. Альтернатива: Отправьте сообщение боту, затем перейдите в браузере по ссылке:
    https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates
    В ответе JSON найдите поле "chat":{"id": ...}.

Шаг 2: Схема подключения

Используем стандартную схему, рассмотренную в предыдущих уроках.

Компонент Пин ESP8266 Примечание
Геркон (Signal) D1 (GPIO5) Режим INPUT_PULLUP
Датчик движения (OUT) D2 (GPIO4) Режим INPUT
DHT11/22 (DATA) D3 (GPIO0) Нужен резистор 4.7-10кОм к VCC
VCC 3V3 / 5V В зависимости от датчика
GND GND Общая земля

Шаг 3: Установка библиотек

В Arduino IDE установите следующие библиотеки:

  1. UniversalTelegramBot (автор: Brian Lough) — для работы с API Telegram.
  2. ArduinoJson (автор: Benoit Blanchon) — версия 6.x (требуется для парсинга ответов).
  3. DHT sensor library (Adafruit) — для датчика температуры.

Программный код

Замените YOUR_BOT_TOKEN и YOUR_CHAT_ID на полученные ранее данные. Также укажите данные вашей Wi-Fi сети.

// --- НАСТРОЙКИ WI-FI ---
const char* ssid = "YOUR_WIFI_SSID";
const char* password = "YOUR_WIFI_PASSWORD";

// --- НАСТРОЙКИ TELEGRAM ---
#define BOTtoken "YOUR_BOT_TOKEN"  // Токен от BotFather
#define BOTchatID "YOUR_CHAT_ID"   // Ваш Chat ID

// --- НАСТРОЙКИ ПИНОВ ---
#define REED_PIN D1      // Геркон
#define MOTION_PIN D2    // Движение
#define DHT_PIN D3       // Температура
#define DHTTYPE DHT11    // Или DHT22

// Инициализация
X509List cert(TELEGRAM_CERTIFICATE_ROOT);
WiFiClientSecure secured_client;
UniversalTelegramBot bot(BOTtoken, secured_client);
DHT dht(DHT_PIN, DHTTYPE);

// Переменные состояния
bool doorOpen = false;
bool motionDetected = false;
unsigned long lastMotionTime = 0;
unsigned long botLastTime = 0;
const unsigned long botRequestDelay = 1000; // Задержка между запросами к боту

void setup() {
  Serial.begin(115200);
  
  // Настройка пинов
  pinMode(REED_PIN, INPUT_PULLUP);
  pinMode(MOTION_PIN, INPUT);
  dht.begin();

  // Подключение к Wi-Fi
  WiFi.mode(WIFI_STA);
  WiFi.begin(ssid, password);
  Serial.print("Connecting to Wifi: ");
  Serial.println(ssid);
  
  while (WiFi.status() != WL_CONNECTED) {
    Serial.print(".");
    delay(500);
  }
  Serial.println("\nWiFi connected");
  
  // Настройка времени для проверки сертификатов
  configTime(0, 0, "pool.ntp.org");
  
  secured_client.setTrustAnchors(&cert);
  
  // Приветственное сообщение
  bot.sendMessage(BOTchatID, "🏠 Система охраны запущена!\nWi-Fi подключен.\nВсе датчики в норме.", "");
}

void loop() {
  // Проверка новых сообщений от бота (опционально, если нужно управление)
  if (millis() - botLastTime > botRequestDelay) {
    // bot.handleUpdate(responses); // Можно раскомментировать для обратной связи
    botLastTime = millis();
  }

  checkSensors();
  delay(200);
}

void checkSensors() {
  // 1. Проверка геркона (Дверь)
  bool currentDoorState = digitalRead(REED_PIN);
  // LOW = закрыто (замкнуто на землю), HIGH = открыто (подтяжка)
  if (currentDoorState == HIGH && !doorOpen) {
    doorOpen = true;
    sendAlarm("🚨 ТРЕВОГА: Дверь открыта!");
  } else if (currentDoorState == LOW && doorOpen) {
    doorOpen = false;
    // Опционально: сообщить о закрытии
    // bot.sendMessage(BOTchatID, "✅ Дверь закрыта", "");
  }

  // 2. Проверка движения
  int motionVal = digitalRead(MOTION_PIN);
  if (motionVal == HIGH) {
    if (!motionDetected) {
      motionDetected = true;
      lastMotionTime = millis();
      float temp = dht.readTemperature();
      String msg = "👁️ Обнаружено движение!\n";
      msg += "🌡️ Температура: " + String(temp, 1) + " *C";
      sendAlarm(msg);
    }
    lastMotionTime = millis(); // Продлеваем таймер
  }
  
  // Сброс флага движения через 5 секунд тишины
  if (motionDetected && (millis() - lastMotionTime > 5000)) {
    motionDetected = false;
  }
}

void sendAlarm(String message) {
  Serial.println("Отправка уведомления: " + message);
  // Экранирование спецсимволов для Markdown не обязательно для простого текста,
  // но лучше использовать режим ParseMode
  bot.sendMessage(BOTchatID, message, "Markdown");
  delay(1000); // Небольшая пауза, чтобы не спамить API при дребезге
}

Разбор ключевых моментов:

  • WiFiClientSecure: Используется для защищенного соединения с серверами Telegram (порт 443).
  • configTime: Необходима для корректной проверки SSL-сертификатов. Убедитесь, что у ESP8266 есть доступ к интернету для синхронизации времени при старте.
  • Логика уведомлений: Сообщения отправляются только при изменении состояния (флаги doorOpen, motionDetected). Это предотвращает спам тысячами сообщений, пока дверь открыта.
  • Температура в тревоге: При срабатывании движения мы сразу считываем температуру и добавляем её в сообщение, чтобы вы понимали контекст (например, это человек или просто сквозняк/нагрев).

Возможные проблемы и решения

  • Ошибка SSL / Connection Failed:
    • Проверьте, правильно ли установлено системное время (функция configTime).
    • Убедитесь, что библиотека UniversalTelegramBot свежая.
    • Попробуйте перезагрузить плату.
  • Сообщения не приходят:
    • Проверьте правильность Token и Chat ID.
    • Убедитесь, что вы нажали Start в диалоге с ботом.
    • Проверьте консоль (Serial Monitor) на наличие ошибок подключения к Wi-Fi.
  • Ложные срабатывания:
    • Для геркона добавьте программную задержку или конденсатор.
    • Для датчика движения уменьшите чувствительность потенциометром.

Куда развивать проект?

Эта система — отличная база. Вот что можно добавить:

  • Команды боту: Реализуйте обработку входящих сообщений. Например, при команде /status бот будет присылать текущую температуру и состояние дверей.
  • Deep Sleep: Для автономной работы от батареи используйте глубокий сон. ESP8266 будет просыпаться только при срабатывании прерывания на пине геркона или движения, отправлять сообщение и снова засыпать.
  • Фото с камеры: Если подключить ESP32-CAM вместо ESP8266, можно отправлять фотографию нарушителя прямо в чат.
  • Сирена: Добавьте реле с сиреной, которое включается автоматически при отправке тревоги.

Заключение

Мы создали полноценную систему безопасности, которая не требует дорогих подписок и облачных сервисов. Все данные идут напрямую с вашего устройства в ваш мессенджер. ESP8266 доказал, что даже недорогой микроконтроллер может стать сердцем умного дома, обеспечивая безопасность и спокойствие.
