Система мониторинга на ESP8266

В этом проекте мы объединим знания из предыдущих статей и создадим полноценную станцию мониторинга. Устройство будет отслеживать температуру, фиксировать открытие двери и обнаруживать движение, выводя всю информацию в реальном времени на OLED-дисплей. Это отличная база для создания умной системы безопасности или метеостанции.

Архитектура проекта

Наша система состоит из микроконтроллера ESP8266, который опрашивает три типа датчиков и выводит данные на экран:

📊 Структура системы
🌡️ Температура Датчик DS18B20
Шина 1-Wire (D4)
🚪 Дверь Магнитный геркон
Digital Input (D1)
🏃 Движение Датчик AM312
Digital Input (D2)
🖥️ Дисплей OLED 128x64
I2C (SDA-D5, SCL-D6)

Необходимые компоненты

  • ESP8266 (NodeMCU или Wemos D1 Mini).
  • OLED-дисплей 0.96" I2C (SSD1306).
  • Цифровой датчик температуры DS18B20 + резистор 4.7 кОм.
  • Датчик движения AM312 (или HC-SR501).
  • Магнитный геркон + магнит.
  • Резистор 10 кОм (для геркона, если не используем INPUT_PULLUP).
  • Макетная плата и провода.

Схема подключения

Так как мы используем много устройств, важно правильно распределить пины, чтобы избежать конфликтов.

Устройство Пин ESP8266 Примечание
OLED VCC 3V3 Питание
OLED GND GND Земля
OLED SCL D1 (GPIO5) I2C Clock
OLED SDA D2 (GPIO4) I2C Data
DS18B20 DATA D4 (GPIO2) Не забудьте резистор 4.7к к 3V3!
Геркон OUT D5 (GPIO14) Режим INPUT_PULLUP
AM312 OUT D6 (GPIO12) Режим INPUT
⚠️ Внимание по пинам В этой схеме мы изменили стандартные пины I2C для OLED (D1/D2), чтобы освободить место для других датчиков, либо использовали стандартные, но убедились, что они не конфликтуют.
В примере кода ниже используются следующие настройки:
  • I2C: SDA=D2, SCL=D1 (Стандарт для многих библиотек ESP8266)
  • DS18B20: D4
  • Геркон: D5
  • Движение: D6
Если у вас другая разводка, просто измените определения в начале кода.

Необходимые библиотеки

Установите следующие библиотеки через Менеджер библиотек Arduino IDE:

  1. Adafruit SSD1306 и Adafruit GFX Library (для дисплея).
  2. OneWire (для шины 1-Wire).
  3. DallasTemperature (для датчика температуры).

Программный код

Код опрашивает все датчики в цикле, обрабатывает события (открытие двери, движение) и обновляет дисплей. Для защиты от дребезга контактов и ложных срабатываний добавлены простые таймеры.

#include 
#include 
#include 
#include 
#include 

// --- НАСТРОЙКИ ПИНОВ ---
#define OLED_SDA D2
#define OLED_SCL D1
#define TEMP_PIN D4
#define REED_PIN D5   // Геркон
#define MOTION_PIN D6 // Движение

// --- ИНИЦИАЛИЗАЦИЯ ОБЪЕКТОВ ---
Adafruit_SSD1306 display(128, 64, &Wire, -1);
OneWire oneWire(TEMP_PIN);
DallasTemperature sensors(&oneWire);

// --- ПЕРЕМЕННЫЕ СОСТОЯНИЯ ---
bool doorOpen = false;
bool motionDetected = false;
unsigned long lastMotionTime = 0;
float temperature = 0.0;

void setup() {
  Serial.begin(115200);
  
  // Настройка пинов
  pinMode(REED_PIN, INPUT_PULLUP); // Внутренняя подтяжка для геркона
  pinMode(MOTION_PIN, INPUT);
  
  // Инициализация дисплея
  if(!display.begin(SSD1306_SWITCHCAPVCC, 0x3C)) {
    Serial.println(F("OLED failed"));
    for(;;);
  }
  display.clearDisplay();
  display.setTextColor(WHITE);
  display.setTextSize(1);
  display.setCursor(0,0);
  display.println("System Boot...");
  display.display();
  delay(1000);
  
  // Инициализация датчика температуры
  sensors.begin();
  
  drawMainScreen();
}

void loop() {
  // 1. Чтение температуры
  sensors.requestTemperatures();
  temperature = sensors.getTempCByIndex(0);
  if (temperature == -127.00) temperature = -99.9; // Ошибка датчика

  // 2. Чтение геркона (Дверь)
  // LOW = замкнут (дверь закрыта), HIGH = разомкнут (дверь открыта) при INPUT_PULLUP
  bool currentDoorState = digitalRead(REED_PIN);
  if (currentDoorState == HIGH && !doorOpen) {
    doorOpen = true;
    showAlarm("DOOR OPEN!");
  } else if (currentDoorState == LOW && doorOpen) {
    doorOpen = false;
    drawMainScreen();
  }

  // 3. Чтение движения
  int motionVal = digitalRead(MOTION_PIN);
  if (motionVal == HIGH) {
    motionDetected = true;
    lastMotionTime = millis();
  }
  // Сброс флага движения, если прошло 3 секунды без новых сигналов
  if (motionDetected && (millis() - lastMotionTime > 3000)) {
    motionDetected = false;
    drawMainScreen();
  }

  // 4. Обновление дисплея, если есть активность
  if (motionDetected || doorOpen) {
    // Экран уже обновлен функциями showAlarm или будет обновлен ниже
    // Для простоты в этом примере мы рисуем статус поверх основного экрана
    updateStatusOverlay();
  } else {
    drawMainScreen();
  }
  
  delay(500);
}

// Функция отрисовки основного экрана
void drawMainScreen() {
  display.clearDisplay();
  display.setTextSize(1);
  display.setCursor(0, 0);
  display.println("--- MONITORING ---");
  
  display.setTextSize(2);
  display.setCursor(0, 20);
  display.print(temperature, 1);
  display.print("*C");
  
  display.setTextSize(1);
  display.setCursor(0, 45);
  display.print("Door: ");
  display.println(doorOpen ? "OPEN" : "Closed");
  
  display.setCursor(0, 55);
  display.print("Motion: ");
  display.println(motionDetected ? "YES" : "No");
  
  display.display();
}

// Функция показа тревоги на весь экран
void showAlarm(String text) {
  display.clearDisplay();
  display.setTextSize(2);
  display.setCursor(0, 20);
  display.println(text);
  display.display();
}

// Обновление статуса поверх основного экрана (мигание иконки)
void updateStatusOverlay() {
  drawMainScreen();
  // Можно добавить мигающий символ тревоги
  if (millis() % 1000 < 500) {
    display.fillRect(120, 0, 8, 8, WHITE);
    display.display();
  }
}

Как это работает:

  1. Инициализация: При старте проверяются все устройства, выводится сообщение о загрузке.
  2. Температура: Считывается каждые полсекунды и отображается крупным шрифтом.
  3. Геркон: Используется внутренняя подтяжка (INPUT_PULLUP). При размыкании (открытии двери) экран очищается и пишет тревогу.
  4. Движение: При обнаружении движения ставится флаг. Если в течение 3 секунд новых сигналов нет, флаг сбрасывается.
  5. Дисплей: Функция drawMainScreen() рисует основной интерфейс, а showAlarm() перекрывает его сообщением о тревоге.

Возможности расширения

Этот проект — отличная основа, которую можно развивать:

  • Wi-Fi уведомления: Добавьте отправку сообщений в Telegram при открытии двери.
  • Веб-сервер: Организуйте доступ к показаниям датчиков через браузер.
  • Логирование: Сохраняйте историю температур и событий во внутреннюю память (SPIFFS) или на SD-карту.
  • Сирена: Подключите активный зуммер к пину D7, чтобы система пищала при тревоге.

Заключение

Мы успешно объединили четыре разных модуля в единую систему. ESP8266 легко справляется с одновременным опросом цифровых и однопроводных датчиков, а OLED-дисплей обеспечивает наглядную визуализацию. Такой блок можно установить на входной двери, в холодильнике или использовать как настольную метеостанцию с функцией охраны.
