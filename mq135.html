Подключение датчика газа MQ135 к ESP8266

Датчик газа MQ135 — это универсальный сенсор качества воздуха, способный обнаруживать широкий спектр вредных газов: аммиак, сернистый газ, бензол, дым и углекислый газ (CO2). В этой статье мы рассмотрим подключение модуля к ESP8266, особенности его прогрева и написание кода для считывания показаний.

Особенности датчика MQ135

Модуль MQ135 содержит чувствительный элемент из диоксида олова (SnO2), сопротивление которого меняется при контакте с загрязненным воздухом. Модуль обычно имеет два типа выходов:

  • Аналоговый выход (AO): Выдает напряжение, пропорциональное концентрации газа. Используется для оценки уровня загрязнения.
  • Цифровой выход (DO): Срабатывает как переключатель при превышении порогового значения (настраивается потенциометром на плате).

Для получения точных данных мы будем использовать аналоговый выход и микроконтроллер ESP8266.

Что вам понадобится

  • Микроконтроллер ESP8266 (NodeMCU, Wemos D1 Mini).
  • Датчик газа MQ135 (модуль с компаратором или сырой сенсор).
  • Соединительные провода.
  • Компьютер с Arduino IDE.
  • USB-кабель.

Схема подключения

Подключение модуля MQ135 к ESP8266 осуществляется через аналоговый вход. Обратите внимание, что датчик потребляет значительный ток для нагрева элемента, поэтому питание лучше брать стабильное.

Пин MQ135 Пин ESP8266 Описание
VCC 5V (рекомендуется) или 3V3 Питание нагревательного элемента
GND GND Общая земля
AO A0 (GPIO17) Аналоговый сигнал
DO Не подключается Цифровой выход (не используется в этом примере)
⚠️ ВАЖНО: Время прогрева Датчик MQ135 требует времени на прогрев чувствительного элемента перед первым использованием.
1. При первом включении дайте датчику поработать от 24 до 48 часов в обычном режиме для стабилизации.
2. Перед каждым измерением рекомендуется давать ему прогреться минимум 1-2 минуты после подачи питания.
3. Не вдыхайте пары от работающего датчика в первые часы работы!

Порядок подключения:

  1. Подключите VCC датчика к пину 5V на NodeMCU (или 3V3, если нет доступа к 5В, но точность может снизиться).
  2. Подключите GND датчика к GND ESP8266.
  3. Подключите вывод AO датчика к аналоговому входу A0 на ESP8266.

Пример кода

Приведенный ниже скетч считывает аналоговое значение, преобразует его в напряжение и выводит условный уровень загрязнения воздуха. Для перевода в ppm (частей на миллион) требуется сложная калибровка по графику из даташита.

// Пины
const int gasPin = A0; // Аналоговый пин подключения датчика

// Переменные
int sensorValue = 0;
float voltage = 0.0;

void setup() {
  Serial.begin(115200);
  Serial.println("Инициализация датчика газа MQ135...");
  Serial.println("Прогрев датчика...");
  
  // Даем датчику немного прогреться при старте
  delay(2000); 
}

void loop() {
  // Считываем значение с аналогового входа (0-1023)
  sensorValue = analogRead(gasPin);
  
  // Преобразуем в напряжение (для ESP8266 диапазон 0-3.3В)
  // Примечание: Если датчик питается от 5В, а вход A0 ограничено 1В или 3.3В,
  // убедитесь, что напряжение на AO не превышает допустимое для ESP.
  // Обычно модули имеют делитель, но лучше перепроверить мультиметром.
  voltage = sensorValue * (3.3 / 1024.0);
  
  Serial.print("Значение ADC: ");
  Serial.print(sensorValue);
  Serial.print(" | Напряжение: ");
  Serial.print(voltage, 3);
  Serial.print(" В");
  
  // Простая логика оценки качества воздуха (требует калибровки!)
  // Чем выше напряжение, тем больше концентрация газа
  if (voltage < 1.0) {
    Serial.println(" | Статус: Чистый воздух");
  } else if (voltage < 2.0) {
    Serial.println(" | Статус: Загрязнение низкое");
  } else if (voltage < 2.8) {
    Serial.println(" | Статус: Загрязнение среднее");
  } else {
    Serial.println(" | Статус: ОПАСНО! Высокая концентрация");
  }
  
  delay(1000);
}
💡 Калибровка для ppm Для получения точных значений в ppm (например, CO2) необходимо:
1. Измерить сопротивление сенсора в чистом воздухе (R0).
2. Использовать формулу из даташита: Rs/R0 = a * (ppm)^b, где коэффициенты a и b берутся из графика чувствительности.
3. В бытовых проектах часто достаточно относительных единиц (ADC value), чтобы отслеживать динамику изменения качества воздуха (проветрили комнату -> показания упали).

Возможные проблемы

  • Показания "скачут": Датчик чувствителен к перепадам напряжения. Попробуйте добавить конденсатор 100мкФ между VCC и GND рядом с датчиком.
  • Значения не меняются: Убедитесь, что вы используете пин AO, а не DO. Проверьте, прогрелся ли датчик (он должен быть теплым).
  • ESP8266 перезагружается: Датчик потребляет ток около 150-200мА для нагрева. Если вы питаете всю схему от слабого USB-порта или нестабильного источника, напряжения может не хватать. Используйте качественный блок питания.

Применение в умном доме

Полученные данные можно использовать для:

  • Автоматического включения вытяжки или вентилятора при ухудшении качества воздуха.
  • Отправки уведомлений в Telegram, если зафиксирован дым или опасная концентрация газов.
  • Ведения логов качества воздуха за сутки/неделю для анализа.
