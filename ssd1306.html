Подключение OLED-дисплея 128×64 к ESP8266

OLED-дисплей на базе контроллера SSD1306 с разрешением 128×64 пикселей — это один из самых популярных способов вывода информации в проектах на ESP8266. Он отличается высокой контрастностью, широким углом обзора, низким энергопотреблением и подключается всего по двум проводам благодаря интерфейсу I2C.

Преимущества дисплея SSD1306

  • Интерфейс I2C: Требует всего 2 провода для передачи данных (SDA и SCK).
  • Самоподсветка: Каждый пиксель светится самостоятельно, черный цвет — это выключенный пиксель (идеальный контраст).
  • Низкое потребление: Потребляет значительно меньше энергии, чем LCD экраны с подсветкой.
  • Компактность: Малые габариты позволяют встраивать его в носимые устройства.

Что вам понадобится

  • Микроконтроллер ESP8266 (NodeMCU, Wemos D1 Mini).
  • OLED-дисплей 0.96" 128x64 (контроллер SSD1306).
  • Соединительные провода.
  • Компьютер с Arduino IDE.

Схема подключения (I2C)

Дисплей работает по шине I2C. На платах ESP8266 (NodeMCU, Wemos) пины для I2C уже определены аппаратно.

Пин OLED Пин ESP8266 (NodeMCU/Wemos) Описание
VCC 3V3 Питание 3.3 В (некоторые модули работают и от 5В, но лучше 3.3В)
GND GND Общая земля
SCL (Clock) D1 (GPIO5) Линия тактирования I2C
SDA (Data) D2 (GPIO4) Линия данных I2C
💡 Примечание по пинам По умолчанию в библиотеках для ESP8266 используются пины:
SDA = GPIO4 (D2)
SCL = GPIO5 (D1)
Если ваш дисплей не работает, проверьте маркировку на плате. Иногда пины могут быть перепутаны местами на дешевых китайских модулях.

Порядок подключения:

  1. Подключите VCC дисплея к 3V3 на ESP8266.
  2. Подключите GND дисплея к GND.
  3. Подключите SCL дисплея к пину D1.
  4. Подключите SDA дисплея к пину D2.

Настройка Arduino IDE

Для работы с дисплеем нам понадобятся две библиотеки. Установите их через Sketch > Include Library > Manage Libraries:

  1. Adafruit SSD1306 — основная библиотека для управления дисплеем.
  2. Adafruit GFX Library — базовая графическая библиотека (зависимость для первой).

После установки, возможно, потребуется открыть файл Adafruit_SSD1306.h и раскомментировать строку с определением разрешения дисплея (#define SSD1306_128_64), если примеры не запускаются.

Пример кода

Этот скетч инициализирует дисплей, выводит приветственный текст, рисует прямоугольник и отображает счетчик секунд.

#include 
#include 
#include 

#define SCREEN_WIDTH 128
#define SCREEN_HEIGHT 64

// Объявление дисплея с адресом 0x3C (стандартный адрес SSD1306)
Adafruit_SSD1306 display(SCREEN_WIDTH, SCREEN_HEIGHT, &Wire, -1);

void setup() {
  Serial.begin(115200);

  // Инициализация дисплея
  if(!display.begin(SSD1306_SWITCHCAPVCC, 0x3C)) {
    Serial.println(F("SSD1306 allocation failed"));
    for(;;); // Зацикливаемся при ошибке
  }

  // Очистка буфера
  display.clearDisplay();
  
  // Настройка текста
  display.setTextSize(1);
  display.setTextColor(WHITE);
  display.setCursor(0, 0);
  
  display.println("Hello, ESP8266!");
  display.println("OLED Test 128x64");
  display.println("----------------");
  display.print("IP: 192.168.1.100");
  
  display.display(); // Отправка буфера на экран
  delay(2000);
}

void loop() {
  display.clearDisplay();
  
  display.setTextSize(2);
  display.setCursor(0, 10);
  display.print("Time: ");
  
  display.setTextSize(3);
  display.setCursor(0, 35);
  display.print(millis() / 1000);
  
  display.drawRect(0, 0, 128, 64, WHITE); // Рамка
  
  display.display();
  delay(1000);
}

Разбор основных команд:

  • display.begin(...): Инициализирует дисплей. Адрес 0x3C является стандартным. Если не работает, попробуйте 0x3D.
  • display.clearDisplay(): Очищает виртуальный буфер (не забудьте вызвать display.display() после рисования, чтобы изменения появились на экране).
  • display.setTextSize(n): Устанавливает размер шрифта (1 — базовый, 2 — в два раза больше и т.д.).
  • display.setCursor(x, y): Устанавливает курсор в координаты X, Y (0,0 — левый верхний угол).
  • display.display(): Критически важная функция! Переносит содержимое буфера на физический экран.

Возможные проблемы

  • Экран черный / мусор:
    • Проверьте подключение проводов SDA и SCL (не перепутаны ли они).
    • Убедитесь, что в коде указан правильный адрес (0x3C или 0x3D). Узнать адрес можно с помощью сканера I2C.
    • Проверьте, закомментировано/раскомментировано ли правильное разрешение в библиотеке.
  • Текст не виден: Возможно, цвет текста установлен в BLACK на черном фоне. Используйте константу WHITE (или INVERSE для инверсии).
  • Артефакты на экране: Попробуйте добавить конденсатор 10мкФ между VCC и GND рядом с дисплеем для стабилизации питания.

Применение в проектах

OLED-дисплей отлично подходит для:

  • Отображения показаний датчиков (температура, влажность, давление) в реальном времени.
  • Создания меню навигации для умного дома.
  • Визуализации графиков и диаграмм.
  • Отображения статуса Wi-Fi соединения и IP-адреса при старте устройства.
